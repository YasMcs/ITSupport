# 4. Documentación del Frontend

## Tecnologías

El frontend del proyecto de soporte TI utiliza las siguientes librerías y herramientas, cada una con un propósito específico:

- **React**: Librería principal para construir la interfaz de usuario de forma declarativa y componer la aplicación en componentes reutilizables.
- **Vite**: Herramienta de construcción moderna que proporciona desarrollo rápido con hot module replacement (HMR) y builds optimizados.
- **Tailwind CSS**: Framework de CSS utility-first para diseñar interfaces responsivas y consistentes sin escribir CSS personalizado extenso.
- **Lucide React**: Conjunto de iconos SVG minimalistas y personalizables para mejorar la UX visual.
- **Sonner**: Librería ligera para notificaciones toast que proporciona feedback inmediato al usuario.
- **Axios**: Cliente HTTP para realizar peticiones al backend con interceptores para manejo automático de autenticación.
- **React Router DOM**: Manejo de rutas en aplicaciones SPA con protección de rutas por rol.
- **@stomp/stompjs y sockjs-client**: Protocolo STOMP sobre WebSocket para comunicación en tiempo real (actualizaciones de tickets y comentarios).

## Estructura del Proyecto

La organización de carpetas sigue un patrón escalable y mantenible:

```
src/
├── components/     # Componentes reutilizables
│   ├── ui/         # Base UI: Button, Input, Table, FilterBar, Badge, etc.
│   ├── layout/     # Navbar, Sidebar, PageWrapper
│   └── tickets/    # Específicos: TicketTable, TicketForm
│   └── usuarios/   # Específicos por módulo
├── pages/          # Vistas principales: TicketsPage, UsersPage, etc.
├── hooks/          # Lógica reutilizable: useAuth, useTickets
├── services/       # API clients: ticketService, authService
├── context/        # Estado global: AuthContext, WebSocketContext
├── utils/          # Helpers: formatDate, security
├── constants/      # ROLES, TICKET_STATUS
└── routes/         # AppRouter, ProtectedRoute
```

**Escalabilidad**: 
- Separación clara por responsabilidades (UI base vs. específica, lógica vs. vista).
- Fácil adición de módulos (nuevo directorio en components/module).
- Estado local por página + global mínimo evita re-renders innecesarios.
- Servicios encapsulan API, facilitando cambios en backend.

## Manejo de Estados y Eventos

Utilizamos **useState** y **useEffect** para lógica local en páginas:

- **useState** para filtros, datos cargados, estados de UI (ej. panel de filtros abierto).
- **useEffect** para cargar datos iniciales al montar la página, subscriptions WebSocket, limpieza.

Ejemplo en TicketsPage: 
- `const [filters, setFilters] = useState({estado: '', prioridad: ''})`
- `const [showFilters, setShowFilters] = useState(false)`
- `useEffect` fetches tickets y aplica filtros locales.
- Filtrado client-side para respuesta inmediata; servidor para grandes datasets.

Eventos: onFilterChange actualiza estado, triggering re-render filtrado. Apertura de panel toggles `showFilters`.

## Consumo de API

El frontend se conecta al backend vía **axios** centralizado en `src/services/api.js`:

- **Token JWT**: Almacenado en sessionStorage, inyectado automáticamente en headers:
  ```
  config.headers.Authorization = `Bearer ${token}`;
  ```
- **Interceptors**: 
  - Request: Agrega token a todas las peticiones excepto login/registro.
  - Response: Detecta 401/403 → limpia sesión, redirige a login.
- **Scoped endpoints**: `ticketService.getScoped(role)` usa `/api/tickets/mis-tickets`, etc.

Ejemplo de interceptor:

```jsx
api.interceptors.request.use((config) => {
  const token = window.sessionStorage.getItem(TOKEN_KEY);
  if (token && !isPublicAuthRoute) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

### 2. Lógica de Clickable Rows en Table

Filas clickeables con accesibilidad (Enter/Space), ignora botones internos:

```jsx
export function Table({ columns, data, onRowClick }) {
  const handleRowClick = (event, row) => {
    if (!isInteractive || shouldIgnoreRowClick(event.target)) return;
    onRowClick(row); // Navega a detalle
  };

  return (
    <table>
      <tbody>
        {data.map((row) => (
          <tr
            onClick={(e) => handleRowClick(e, row)}
            onKeyDown={(e) => handleRowKeyDown(e, row)}
            tabIndex={0} role=\"button\"
            className=\"cursor-pointer hover:bg-white/[0.05]\"
          >
            {/* celdas */}
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```
